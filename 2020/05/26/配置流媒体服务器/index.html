搭建

环境

系统环境：deepin 15.11

nginx 版本：1.17

ffmpeg 版本 ：3.2.12-1~deb9u1

一开始也是用的windwos ，有次在哔哩哔哩上看见一个deepin 系统展示视频，然后就入坑了。用了一段时间之后觉得，还是很不错的，开发软件什么的都还可以支持（hbulderx 就算了，强装过，然后用哭了）

安装

  1. 安装nginx 编译相关依赖库

    安装gcc g++的依赖库

    sudo apt-get install build-essential
sudo apt-get install libtool

安装pcre依赖库（http://www.pcre.org/）

   sudo apt-get install libpcre3 libpcre3-dev

安装zlib依赖库（http://www.zlib.net）

   sudo apt-get install zlib1g-dev

安装SSL依赖库

   sudo apt-get install openssl

  1. nginx 下载

    nginx官方下载页面：http://nginx.org/en/download.html
    configure配置文件详解：http://nginx.org/en/docs/configure.html

  1. nginx-rtmp-module 下载

    官方下载

此文件是nginx中的一个模块，用于音视频的点播、直播。也是搭建流媒体服务器的核心模块

  1. 编译安装

    # 解压下载好的nginx 压缩包
tar zxvf nginx-1.17.10.tar.gz

# 新建nginx 编译目录nginx
# module 用于存放模块文件 
# myapp 用于存放视频数据
# vod 用于存放点播视频数据
mkdir -p nginx/module
mkdir -p nginx/myapp
mkdir -p nginx/vod/flvs

# 进入到module目录
cd 自定义路径/nginx/module

# 解压下载的nginx-rtmp-module.zip到module中
unzip nginx-rtmp-module-master.zip

# 返回nginx-1.17.10 解压目录，执行如下语句
./configure --prefix=自定义路径/nginx --add-module=自定义路径/nginx/module/nginx-rtmp-module --conf-path=自定义路径/nginx/nginx.conf

# 编译并安装
make && make install

  1. 验证启动

    # 使用默认配置文件启动nginx
sudo 自定义路径/nginx/sbin/nginx -c 自定义路径/nginx/conf/nginx.conf

# 查看nginx的版本号
sudo 自定义路径/nginx/sbin/nginx -v

    浏览器访问localhost，默认80端口，出现如下界面表示nginx安装成功:

    注意:如果启动后提示403 ,可以尝试修改nginx/conf/nginx.conf 中第一行语句

    # user nobody;

## 改动为
user root;

  1. 安装ffmpeg

    参考百度经验

  1. 修改配置文件

    将/usr/loca/nginx/nginx.conf

    user root;
worker_processes  1;

error_log  logs/error.log debug;

events {
    worker_connections  1024;
}

rtmp {
    server {
        listen 1935;

        application myapp {
            live on;

            hls on;
            hls_path 自定义路径/nginx/myapp; #视频存放路径

            #record keyframes;
            #record_path /tmp;
            #record_max_size 128K;
            #record_interval 30s;
            #record_suffix .this.is.flv;

            #on_publish http://localhost:8080/publish;
            #on_play http://localhost:8080/play;
            #on_record_done http://localhost:8080/record_done;
        }

        ##点播配置
        application vod{
            play 自定义路径/nginx/vod/flvs;
        }
    }
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    #log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
    #                  '$status $body_bytes_sent "$http_referer" '
    #                  '"$http_user_agent" "$http_x_forwarded_for"';

    #access_log  logs/access.log  main;

    sendfile        on;
    #tcp_nopush     on;

    #keepalive_timeout  0;
    keepalive_timeout  65;

    #gzip  on;

    server {
        listen      80;

        location /stat {
            rtmp_stat all;
            rtmp_stat_stylesheet stat.xsl;
        }

        location /stat.xsl {
            root 自定义路径/nginx/module/nginx-rtmp-module-master;
        }

        location /control {
            rtmp_control all;
        }

        #location /publish {
        #    return 201;
        #}

        #location /play {
        #    return 202;
        #}

        #location /record_done {
        #    return 203;
        #}

        location /rtmp-publisher {
            root 自定义路径/nginx/module/nginx-rtmp-module-master/test/rtmp-publisher;
        }

        location / {
            root 自定义路径/nginx/module/nginx-rtmp-module-master/test/www;
            index  index.html index.htm;
        }
    }
}

  1. 重启启动nginx

    ## 关闭之前的nginx 服务
sudo pkill nginx

## 启动nginx
sudo ./自定义路径/nginx/sbin/nginx

测试

准备一个视频文件，我这里使用的是deepin官方自带的MP4演示

准备一个收流软件，我这里使用的是vlc

打开vlc 点击媒体—-打开捕获设备

选择网络选项卡，输入rtmp://127.0.0.1:1935/myapp

## 执行推流命令
ffmpeg -re -i 本地视频地址 -c copy -f flv rtmp://127.0.0.1:1935/myapp

如果想测试摄像头视频流将本地视频地址换成摄像头视频流地址即可
